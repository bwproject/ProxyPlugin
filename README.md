# ProxyPlugin

**SOCKS5 proxy routing for Minecraft Paper servers**

Позволяет направлять сетевые запросы плагинов (например, к Maven, GitHub, JitPack) через SOCKS5-прокси **без VPN на весь сервер**.

---

## 📌 Зачем это нужно

На серверах, расположенных в регионах с ограничениями (например, Россия), часто возникают проблемы:

- ❌ плагины не могут скачать зависимости из Maven  
- ❌ ошибки при старте (`Failed to download dependency`)  
- ❌ недоступны GitHub / JitPack / Paper repos  

**ProxyPlugin решает это точечно:**

- направляет только нужные домены через SOCKS5  
- остальной трафик остаётся прямым  
- не требует VPN или изменения сетевой конфигурации сервера  

---

## ⚙️ Как это работает

Плагин:

1. Загружается на этапе `STARTUP` (максимально рано)  
2. Устанавливает глобальный `ProxySelector` в JVM  
3. Перехватывает исходящие HTTP(S) соединения  
4. Проверяет домен запроса  
5. Если домен совпадает с конфигом → отправляет через SOCKS5  

```
Plugin → HTTP request → ProxySelector → SOCKS5 → Интернет
```

---

## ✅ Поддерживается

Работает с библиотеками, использующими стандартный Java networking:

- `HttpURLConnection`  
- `URLConnection`  
- часть Maven Resolver / Aether  
- многие Paper/Spigot плагины  

---

## ⚠️ Ограничения

Плагин **не может перехватить всё**:

- ❌ OkHttp (если прокси задан вручную)  
- ❌ Apache HttpClient с кастомной конфигурацией  
- ❌ нативные или JNI-сетевые библиотеки  

---

## 📦 Установка

1. Скачайте `.jar` из **Releases**  
2. Поместите в папку:

```
/plugins/ProxyPlugin.jar
```

3. Запустите сервер  
4. Отредактируйте `config.yml`  
5. Перезапустите сервер  

---

## 🧾 Конфигурация

```yaml
proxy:
  host: "127.0.0.1"
  port: 1080
  username: "user"
  password: "pass"

domains:
  - "repo.maven.apache.org"
  - "repo.papermc.io"
  - "libraries.minecraft.net"
  - "github"
  - "jitpack"

debug: true
```

---

### 🔹 proxy

| Поле     | Описание              |
|----------|----------------------|
| host     | IP или домен SOCKS5  |
| port     | порт                 |
| username | логин (опционально)  |
| password | пароль (опционально) |

---

### 🔹 domains

Список доменов, которые будут идти через прокси.

Поддерживается простое совпадение:

```
repo.maven.apache.org → OK
maven → поймает CDN домены
```

---

### 🔹 debug

```yaml
debug: true
```

Логи:

```
[PROXY] repo.maven.apache.org
[DIRECT] example.com
```

---

## 🧪 Проверка работы

После запуска сервера в логах должно быть:

```
[ProxyPlugin] [PROXY] repo.maven.apache.org
```

Если видите `[DIRECT]` → домен не совпал.

## ❗ Важные замечания

- Плагин работает **на уровне JVM**, а не сети  
- Не влияет на игровой трафик игроков  
- Не требует root / iptables / VPN  
- Может не успеть перехватить ранние загрузки некоторых плагинов  

---

## 🔮 Планы

- Java Agent версия (100% перехват)  
- wildcard домены (`*.domain.com`)  
- команда `/proxy reload`  
- лог в файл  
- fallback прокси
